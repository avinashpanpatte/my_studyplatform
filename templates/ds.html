Chapter 1: Arrays

1.1 Introduction to Arrays

An array is a fundamental data structure that stores a collection of elements of the same data type in contiguous memory locations. Arrays provide a way to group related data items under a single name for easy access and manipulation.

1.2 Definition

An array is defined as a finite ordered collection of homogeneous data elements stored in contiguous memory locations. Each element in the array can be accessed using an index, which represents its position in the array.

1.3 Types of Arrays

  • One-Dimensional Arrays: Linear arrays that store elements in a single row.
  • Two-Dimensional Arrays: Arrays arranged in rows and columns, like a matrix.
  • Multi-Dimensional Arrays: Arrays with more than two dimensions.

1.4 Memory Representation

In C programming, arrays are stored in contiguous memory locations. For a one-dimensional array of size n, the memory address of element at index i can be calculated as:

Address of arr[i] = Base Address + (i × Size of each element)

1.5 Array Operations

  • Traversal: Visiting each element of the array exactly once.
  • Insertion: Adding a new element at a specified position.
  • Deletion: Removing an element from a specified position.
  • Searching: Finding the location of a particular element.
  • Sorting: Arranging elements in a specific order.

1.6 Advantages and Disadvantages

Advantages:

  • Random access to elements using index (O(1) time)
  • Easy to implement and understand
  • Memory efficient for storing homogeneous data

Disadvantages:

  • Fixed size - cannot be resized dynamically
  • Insertion and deletion operations are expensive (O(n) time)
  • Waste of memory if not fully utilized

1.7 Time Complexity

  • Access: O(1)
  • Search: O(n)
  • Insertion/Deletion: O(n)

1.8 Code Examples

Declaration and Initialization:

int arr[5] = {1, 2, 3, 4, 5};
int arr2[3][3] = {{1,2,3}, {4,5,6}, {7,8,9}};

Traversal:

for(int i = 0; i < 5; i++) {
    printf("%d ", arr[i]);
}

1.9 Applications

  • Storing marks of students in a class
  • Implementing matrices for mathematical operations
  • Lookup tables in compilers
  • Image processing (pixel arrays)

Practice Exercises:

  • Find maximum element in array
  • Rotate array by k positions
  • Find missing number in 1 to n
  • Two sum problem (LeetCode)

Chapter 2: Linked Lists

2.1 Introduction to Linked Lists

A linked list is a linear data structure where elements are stored in nodes, and each node points to the next node in the sequence. Unlike arrays, linked lists do not require contiguous memory allocation.

2.2 Types of Linked Lists

  • Singly Linked List: Each node has data and a pointer to the next node.
  • Doubly Linked List: Each node has data, a pointer to the next node, and a pointer to the previous node.
  • Circular Linked List: The last node points back to the first node, forming a circle.

2.3 Node Structure

In C, a node in a singly linked list can be represented as:

struct Node {
    int data;
    struct Node* next;
};

2.4 Basic Operations

  • Insertion: Adding a node at the beginning, end, or specific position.
  • Deletion: Removing a node from the beginning, end, or specific position.
  • Traversal: Visiting each node in the list.
  • Search: Finding a particular element in the list.
  • Reverse: Reversing the order of nodes in the list.

2.5 Advantages over Arrays

  • Dynamic size - can grow or shrink as needed
  • Efficient insertion and deletion operations
  • No memory wastage due to fixed size
  • Easier to implement certain operations like merging

2.6 Disadvantages

  • No random access - must traverse from the beginning
  • Extra memory for storing pointers
  • Not cache-friendly due to non-contiguous memory

2.7 Time Complexity

  • Access: O(n)
  • Search: O(n)
  • Insertion/Deletion at beginning: O(1)
  • Insertion/Deletion at end: O(n) for singly linked list, O(1) with tail pointer

2.8 Applications

  • Implementation of stacks and queues
  • Dynamic memory allocation
  • Symbol table management in compilers
  • Undo functionality in text editors

2.9 Cycle Detection

A cycle in a linked list occurs when a node points back to a previous node, creating a loop. Floyd's Cycle Detection Algorithm (Tortoise and Hare) can detect cycles in O(n) time and O(1) space.

Practice Exercises:

  • Reverse a linked list
  • Detect and remove cycle

Chapter 3: Stacks & Queues

3.1 Introduction to Stacks

A stack is a linear data structure that follows the Last-In-First-Out (LIFO) principle. The last element added to the stack is the first one to be removed.

3.2 Stack Operations

  • Push: Add an element to the top of the stack
  • Pop: Remove the top element from the stack
  • Peek/Top: View the top element without removing it
  • isEmpty: Check if the stack is empty
  • isFull: Check if the stack is full (for array implementation)

3.3 Stack Implementation

  • Array-based: Fixed size, efficient for small stacks
  • Linked List-based: Dynamic size, no overflow issues

3.4 Introduction to Queues

A queue is a linear data structure that follows the First-In-First-Out (FIFO) principle. The first element added to the queue is the first one to be removed.

3.5 Queue Operations

  • Enqueue: Add an element to the rear of the queue
  • Dequeue: Remove the front element from the queue
  • Front: View the front element without removing it
  • Rear: View the rear element
  • isEmpty: Check if the queue is empty
  • isFull: Check if the queue is full

3.6 Types of Queues

  • Linear Queue: Standard FIFO queue
  • Circular Queue: Last position connects back to first, efficient space usage
  • Priority Queue: Elements dequeued based on priority
  • Deque (Double-ended Queue): Insertion and deletion from both ends

3.7 Applications

Stacks:

  • Function call management (call stack)
  • Expression evaluation and conversion
  • Undo operations in text editors
  • Browser back button functionality

Queues:

  • Process scheduling in operating systems
  • Print job management
  • Breadth-First Search (BFS) algorithm
  • CPU scheduling algorithms

3.8 Time Complexity

  • Stack operations: O(1)
  • Queue operations: O(1)

3.9 Recursion

Recursion is a programming technique where a function calls itself. Every recursive function must have a base case to prevent infinite recursion. Recursion uses a call stack internally.

Examples: Factorial calculation, Fibonacci series, Tower of Hanoi

Practice Exercises:

  • Valid parentheses problem
  • Implement stack using queues
  • Circular queue implementation
  • Next greater element

Chapter 4: Trees

4.1 Introduction to Trees

A tree is a hierarchical data structure consisting of nodes connected by edges. It is a non-linear data structure with a root node and child nodes forming a parent-child relationship.

4.2 Tree Terminology

  • Root: The topmost node of the tree
  • Node: Each element in the tree
  • Edge: Connection between two nodes
  • Parent: Node that has child nodes
  • Child: Node that has a parent
  • Leaf: Node with no children
  • Height: Number of edges on the longest path from root to leaf
  • Depth: Number of edges from root to the node
  • Level: Nodes at the same depth

4.3 Types of Trees

  • Binary Tree: Each node has at most two children
  • Binary Search Tree (BST): Left subtree < root < right subtree
  • AVL Tree: Self-balancing BST with height difference ≤ 1
  • B-Tree: Self-balancing search tree for disk-based storage
  • Red-Black Tree: Self-balancing BST with color properties

4.4 Tree Traversals

  • Inorder (Left-Root-Right): Used in BST to get sorted order
  • Preorder (Root-Left-Right): Used for copying trees
  • Postorder (Left-Right-Root): Used for deleting trees
  • Level Order: Breadth-first traversal using queue

4.5 Binary Search Tree (BST)

A BST is a binary tree where for each node, all elements in its left subtree are less than the node, and all elements in its right subtree are greater than the node.

Operations:

  • Search: O(log n) average case
  • Insert: O(log n) average case
  • Delete: O(log n) average case

4.6 AVL Trees

AVL trees are height-balanced BSTs. The height difference between left and right subtrees is at most 1. Rotations are used to maintain balance: left rotation, right rotation, left-right rotation, right-left rotation.

4.7 Tree Operations

  • Insertion: Add a new node to the tree
  • Deletion: Remove a node from the tree
  • Search: Find a particular node
  • Height: Calculate the height of the tree
  • Diameter: Longest path between any two nodes
  • Lowest Common Ancestor (LCA): Find the lowest common ancestor of two nodes

4.8 Applications

  • File system organization
  • Database indexing (B-Trees)
  • Expression parsing
  • Decision trees in machine learning
  • Huffman coding for data compression

4.9 Time Complexity

  • Traversal: O(n)
  • Search/Insert/Delete in BST: O(log n) average, O(n) worst case
  • Balanced trees: O(log n) for all operations

Practice Exercises:

  • Binary tree traversals
  • Check if BST
  • Lowest common ancestor
  • Serialize/deserialize tree

Chapter 5: Graphs

5.1 Introduction to Graphs

A graph is a non-linear data structure consisting of vertices (nodes) connected by edges. Graphs are used to represent relationships between objects.

5.2 Graph Terminology

  • Vertex/Node: Fundamental unit of a graph
  • Edge: Connection between two vertices
  • Adjacent vertices: Vertices connected by an edge
  • Degree: Number of edges connected to a vertex
  • Path: Sequence of vertices connected by edges
  • Cycle: Path that starts and ends at the same vertex
  • Connected Graph: Path exists between every pair of vertices
  • Weighted Graph: Edges have weights/costs
  • Directed Graph: Edges have direction

5.3 Graph Representations

  • Adjacency Matrix: 2D array where matrix[i][j] = 1 if edge exists, 0 otherwise. Space: O(V²)
  • Adjacency List: Array of lists where each list contains adjacent vertices. Space: O(V + E)

5.4 Graph Traversal Algorithms

Depth-First Search (DFS): Explores as far as possible along each branch before backtracking. Uses stack (recursion or explicit stack).

Breadth-First Search (BFS): Explores all vertices at the present depth before moving to next depth level. Uses queue.

5.5 Shortest Path Algorithms

  • Dijkstra's Algorithm: Finds shortest path from source to all vertices in weighted graph with non-negative weights
  • Bellman-Ford Algorithm: Handles negative weights, detects negative cycles
  • Floyd-Warshall Algorithm: Finds shortest paths between all pairs of vertices

5.6 Minimum Spanning Tree (MST)

A spanning tree connects all vertices with minimum total edge weight and no cycles.

  • Kruskal's Algorithm: Greedy approach, sorts edges by weight
  • Prim's Algorithm: Greedy approach, grows tree from a starting vertex

5.7 Topological Sorting

Linear ordering of vertices in a directed acyclic graph (DAG) such that for every directed edge u→v, vertex u comes before v. Used in task scheduling, dependency resolution.

5.8 Cycle Detection

  • Undirected Graph: Use DFS or Union-Find
  • Directed Graph: Use DFS with color coding (white, gray, black)

5.9 Applications

  • Social networks (Facebook, LinkedIn)
  • Transportation networks (maps, GPS)
  • Computer networks (routing algorithms)
  • Dependency graphs in software
  • Recommendation systems

5.10 Time Complexity

  • DFS/BFS: O(V + E)
  • Dijkstra: O((V + E) log V) with binary heap
  • Kruskal/Prim: O(E log E)
  • Floyd-Warshall: O(V³)

Practice Exercises:

  • Graph traversal (DFS/BFS)
  • Detect cycle in graph
  • Topological sorting
  • Find shortest path

Chapter 6: Sorting & Searching

6.1 Introduction to Sorting

Sorting is the process of arranging elements in a specific order (ascending or descending). Efficient sorting is crucial for many algorithms and data processing tasks.

6.2 Sorting Algorithms

Bubble Sort: Repeatedly swaps adjacent elements if they are in wrong order. Time: O(n²), Space: O(1)

Selection Sort: Finds minimum element and places it at beginning. Time: O(n²), Space: O(1)

Insertion Sort: Builds sorted array one element at a time. Time: O(n²), Space: O(1)

Merge Sort: Divide and conquer algorithm. Time: O(n log n), Space: O(n)

Quick Sort: Partition-based divide and conquer. Time: O(n log n) average, O(n²) worst, Space: O(log n)

Heap Sort: Uses heap data structure. Time: O(n log n), Space: O(1)

6.3 Searching Algorithms

Linear Search: Sequential search through array. Time: O(n)

Binary Search: Divide and conquer on sorted array. Time: O(log n), Space: O(1)

Requirements: Array must be sorted

6.4 Hashing

Hashing maps data to fixed-size values using hash functions. Used in hash tables for O(1) average time complexity for search, insert, delete operations.

Hash Function: Converts key to index

Collision Resolution: Chaining, Open Addressing (Linear Probing, Quadratic Probing, Double Hashing)

Load Factor: Ratio of stored elements to table size

6.5 Complexity Analysis

Time Complexity: How execution time grows with input size

  • Big O (O): Upper bound, worst case
  • Big Ω (Ω): Lower bound, best case
  • Big Θ (Θ): Tight bound, average case

Space Complexity: Memory usage analysis

6.7 Applications

  • Database sorting and searching
  • Search engines (indexing)

6.8 Recursion in Algorithms

Many algorithms use recursion: Merge Sort, Quick Sort, Binary Search, Tree traversals. Recursion requires base case and recursive case.
